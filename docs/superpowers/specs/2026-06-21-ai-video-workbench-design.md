# AI 短视频批量生产工作台设计稿

日期：2026-06-21

## 目标用户

首发服务国内短视频代运营、本地生活商家、剪辑公司、矩阵号团队。第一版聚焦抖音竖屏短视频批量生产。

## 产品形态

- 普通用户只使用 Windows 桌面客户端。
- 云端负责 AI、剪辑、字幕、混剪、积分扣费和任务队列。
- 运营方使用网页运营后台管理用户、积分、模板、任务和样本库。
- 官网只做产品介绍、Windows 下载、申请试用、协议和联系方式。

## MVP 范围

包含：

- 邮箱 + 密码登录
- 邀请码注册
- 团队空间 workspace
- 客户/品牌档案
- 团队素材库
- 抖音链接合规导入 + 粘贴兜底
- 系统爆款库 + 客户私有样本库
- 行业模板库
- AI 爆款文案生成
- 用户确认脚本后再剪辑
- 素材自动打标签 + 用户修正
- 可选 AI 配音
- 自动字幕
- 模板化自动剪辑
- 批量混剪
- 云端任务队列
- MP4 预览与下载
- 积分冻结/扣除/失败退回
- 最小运营后台
- Windows 客户端自动更新

不包含：

- 第三方插件市场
- 完整时间线编辑器
- 设备绑定/机器码授权
- 自动支付
- 数字人主功能
- AI 生图/生视频主功能
- 声音克隆主功能
- 离线模式
- 网页用户端

## 核心架构

- Windows 客户端：Tauri + React + TypeScript
- 后台：Django + PostgreSQL
- 队列/缓存：Redis，后续可升级托管 RabbitMQ/RocketMQ
- Worker：Python Worker + FFmpeg
- 存储：阿里云 OSS
- 云部署：阿里云，API 可用 SAE，Worker 用 ECS 按量实例 + 弹性伸缩
- AI：统一 provider 接口，首发接火山，后续可接其他模型

## AI provider 设计

统一 provider 类型：

- llm_provider：文案/脚本
- tts_provider：配音
- asr_provider：字幕识别
- vision_provider：素材识别/标签
- video_provider：后续数字人、生图、生视频

后台可配置：

- 模型名称
- API Key
- 能力类型
- 是否启用
- 积分价格系数
- 默认模型

用户可在高级设置里选择模型。不同模型允许不同积分价格。API Key 只保存在云端。

## 积分规则

- 用户人工充值，后台加积分。
- 提交任务时冻结积分。
- 成功后正式扣除。
- 系统失败自动退回。
- 用户不满意重新生成，需要重新扣积分。
- 不同任务类型和模型消耗不同积分。

## 任务并发与扩容

- 所有重任务进入队列，不在 API 请求里直接执行。
- 队列按任务类型拆分：文案、字幕、配音、剪辑、导出。
- Worker 无状态，可横向扩容。
- 团队并发数可配置。
- 高峰允许排队。
- 客户端展示排队状态、当前步骤和预计等待时间。
- 架构目标是人多变慢但不崩。

## 文件与保存策略

- 客户端通过云端签名直传 OSS，避免业务服务器承载大文件上传。
- 原始素材默认保存 30 天。
- 成片默认保存 90 天。
- 用户可手动删除。
- 高级套餐后续可延长保存时间。

## 导出规格

第一版只支持：

- 抖音竖屏 9:16
- 1080x1920
- MP4
- 15 秒 / 30 秒 / 60 秒
- 默认烧录字幕
- 自动生成封面

## 批量混剪定义

- 同一批素材生成多条视频。
- 每条视频使用不同文案/开头钩子。
- 按素材标签规则组合并随机排序。
- 字幕、标题、封面、配乐可变化。
- 先不做复杂查重算法或平台查重模拟。

## 素材与客户档案

客户/品牌档案字段：

- 名称
- 行业
- 产品/服务
- 目标人群
- 核心卖点
- 禁用词/不能说的话
- 常用联系方式/引流话术
- 文案风格偏好
- Logo/常用素材

素材标签第一版：

- 产品
- 人物
- 环境
- 价格
- 过程
- 对比
- 证书/资质
- 门头/店铺
- 细节特写

## 抖音链接导入规则

第一版只做合规导入：

- 保存原始链接
- 保存标题/文案
- 保存结构分析
- 保存行业标签
- 保存爆点标签
- 保存改写版本

不默认保存别人的原视频文件。提取失败时让用户粘贴文案兜底。

## 官网/下载页设计

视觉方向：AI 视频生产发布会风。借鉴科技发布会质感，不复制任何具体品牌元素。

色彩：

- 深空黑：#050812
- 星际蓝：#0B5CFF
- 电光青：#00E5FF
- 紫色能量光：#7A5CFF
- 银白文字：#EAF0FF
- 暗灰面板：#111827

首页结构：

1. Hero：AI 短视频批量生产工作台
2. 核心流程：上传素材、生成脚本、云端剪辑、批量导出
3. 功能矩阵：爆款文案、行业模板、字幕、配音、混剪、素材库、任务队列、积分计费
4. 客户场景：代运营、本地生活商家、剪辑公司、矩阵号团队
5. 下载/试用：Windows 客户端下载、联系客服、邀请码注册说明

记忆点：首屏展示“AI 视频生产引擎核心”，素材流进入，中间脚本/字幕/剪辑引擎处理，右侧批量生成抖音视频。

## Windows 客户端页面

第一版 3 个主工作区：

1. 工作台首页：积分、最近项目、任务、快捷入口、公告。
2. 项目创建流程：客户/品牌、行业模板、素材、爆款样本、AI 脚本、确认剪辑。
3. 任务与预览页：排队/处理/完成/失败、当前步骤、预计等待、MP4 预览、下载、重新生成。

客户端保持深色科技工作台风格，但优先保证操作清楚。

## Penpot 用法

Penpot 只作为设计和原型展示工具：

- 画官网/客户端原型
- 管理颜色、字体、组件
- 展示交互流程
- 给开发提供标注和资源

不把 Penpot 并入产品代码，不魔改 Penpot。
